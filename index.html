01 / Foundations

Color palette

KLJ orange is the signature brand color, but it must be darkened for text. Bright #DD5900 carries logos, icons, borders, and large accents, while the darker #C44F00 handles links, buttons, and small text so everything clears WCAG AA. Slate and neutral grays carry most of the interface, with teal as a supporting accent.

02 / SharePoint

Theming SharePoint with the KLJ palette

A SharePoint Online theme is a full Fluent UI color palette, generated from three brand inputs, added to the tenant, and then read automatically by every link, button, and web part. This is the exact path a developer follows to hook the KLJ colors into SharePoint.

Accessibility guardrail: seed the theme with the darker orange themePrimary drives link text and button labels at body size, so it must meet WCAG AA (4.5:1). KLJ Orange #dd5900 is only 3.8:1 on white and fails, so it cannot be the theme primary. Seed the theme with #c44f00 (4.7:1) instead. Bright #dd5900 still lives in the brand, on logos, icons, borders, and large accents, but those never flow through themePrimary, so they stay safe.

1 Pick the three seed colors

Enter these into the Fluent UI Theme Designer. It expands them into the full slot ramp in step 2.

3 Apply it with PowerShell

Add the palette to the tenant theme gallery once. Site owners then select “KLJ Brand” under Settings → Change the look → Theme.

# 1 · Connect to the SharePoint admin center
Connect-SPOService -Url https://kljeng-admin.sharepoint.com

# 2 · KLJ palette (Fluent UI Theme Designer output)
#      Primary #c44f00 · Text #323e48 · Background #ffffff
$klj = @{
  "themePrimary"         = "#c44f00"; "themeLighterAlt"      = "#fdf7f2";
  "themeLighter"         = "#f6dcc7"; "themeLight"           = "#ecbb98";
  "themeTertiary"        = "#da8047"; "themeSecondary"       = "#ca5e12";
  "themeDarkAlt"         = "#b04700"; "themeDark"            = "#953c00";
  "themeDarker"          = "#6e2c00"; "neutralLighterAlt"    = "#f8f8f7";
  "neutralLighter"       = "#f4f4f2"; "neutralLight"         = "#ecece9";
  "neutralQuaternaryAlt" = "#dadad7"; "neutralQuaternary"    = "#d0d0cd";
  "neutralTertiaryAlt"   = "#c8c8c5"; "neutralTertiary"      = "#9a9a9a";
  "neutralSecondary"     = "#666d70"; "neutralPrimaryAlt"    = "#444f58";
  "neutralPrimary"       = "#323e48"; "neutralDark"          = "#2a343d";
  "black"                = "#1b2127"; "white"                = "#ffffff";
  "primaryBackground"    = "#ffffff"; "primaryText"          = "#323e48";
  "bodyBackground"       = "#ffffff"; "bodyText"             = "#323e48";
}

# 3 · Register the theme in the tenant gallery
Add-SPOTheme -Name "KLJ Brand" -Palette $klj -IsInverted $false -Overwrite

4 Reference it in custom code (SPFx)

Bind component styles to theme slots so custom web parts restyle automatically when the KLJ theme is applied. Out-of-the-box links and buttons already inherit themePrimary through Fluent’s semanticColors.link, so no extra CSS is needed for them.

// SPFx web part .module.scss — bind to theme slots, not hex.
// The component then follows whatever theme the site has applied.
.cta {
  color: "[theme: white, default: #ffffff]";
  background-color: "[theme: themePrimary, default: #c44f00]";
}
.cta:hover { background-color: "[theme: themeDarkAlt, default: #b04700]"; }

.link    { color: "[theme: themePrimary, default: #c44f00]"; }
.heading { color: "[theme: neutralPrimary, default: #323e48]"; }
Authoring tip: Apply the theme at the hub site so every associated site inherits it. Set the site header background to #323e48 (Slate) with the white KLJ logo, and reserve full-orange section backgrounds for rare, high-impact moments.

03 / Foundations

Typography

The portal uses Segoe UI, the native SharePoint Online typeface. Standardizing on it means zero web-font loading, a perfect match with the SharePoint chrome and suite bar, and consistent rendering across Microsoft 365. Use Semibold for headings and Regular for body copy. The fallback stack covers non-Windows devices with the closest system font.

04 / Accessibility

Color contrast (WCAG AA)

KLJ Orange #dd5900 is a bright accent: it measures 3.8:1 on white, which clears AA for large text and UI elements but not for normal body text or small labels. To stay compliant, use the darker #c44f00 for any orange text at body size, and reserve #dd5900 for headings, icons, borders, and accents.

Rules of thumb

✓ Do

  • Use #c44f00 for orange links and small text on white.
  • Use white text on #c44f00 (not #dd5900) for filled buttons.
  • Use bright #dd5900 for headings, icons, borders, and the topbar accent.
  • Pair Amber #ff9e1b with slate, never as text on white.
  • Set the SharePoint themePrimary slot to #c44f00.

✕ Don't

  • Don't set body text or small labels in #dd5900 on white.
  • Don't put white text on a #dd5900 fill at normal size.
  • Don't use Amber for text on a light background.
  • Don't rely on color alone to signal status. Pair it with a label or icon.
  • Don't drop the visible focus ring on links and buttons.
Target: WCAG 2.1 Level AA. Normal text needs 4.5:1, large text (24px, or 18.66px bold) and UI components need 3:1. Verify any new pairing in a contrast checker such as WebAIM before publishing a page.
